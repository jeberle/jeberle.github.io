They Got Me, I Cut Over from Mercurial to Git

There was a certain inevitability about this moment. I’m a big fan of C-based tools and the Unix philosophy: do one thing well. Git exemplifies this. Mercurial? Not really. Still, I had my reasons to resist.

  • Familiarity. After progressing through CVS and Subversion, the command set in Mercurial was a no-brainer. When it gets right down to it, how much of your mental energy do you want to spend on version control? Not very much right? Mercurial then, dammit.
  • It’s Python. Python is my favorite “pretty darn good for all sorts of problems” language. A DVCS in Python, hey why not?
  • The Gibibyte thing. For some reason, Linux has embraced the intellectually bankrupt notion of the kibi-, mebi-, gibi-byte thing. I really hate this. The fact that every Git pull or push sticks this in your face is really annoying.

I first tried forking and modifying the output to remedy item 3. This seemed a bit extreme, and instead I’ve just learned to avert my eyes and fight back my gag reflex.

Having made this step, I can start enjoying Git’s advantages:

  • Speed. Speed is addictive. You don’t go back from speed.
  • It’s a file system within a file system. You can do whatever you want with it. It’s your stuff after all. This just feels more natural than forever appending to a “I did this” log.

Like I said, this was inevitable.
