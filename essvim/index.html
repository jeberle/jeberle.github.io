Improve Your Vim

Follow the 10 edicts below to Improve Your Vim.

Note: I to use modern terms in place of Vim terms throughout this doc.

Modern Term Vi Term
Cut :delete
Copy :yank
Paste :put

Break Free

Before we get going, it’s actually pretty easy to get things wedged, so it’s important to know how to respond.

Command Description
<CapsLock> CapsLock is Vi’s kryptonite — never use it, check if all seems awry
u Undo
<Esc> Get out of Insert, Change, or Append mode
:vi <Enter> Get out of inadvertant Ex Cmdline mode Q and back into Visual mode
:q <Enter> Get out of inadvertant history edit q:
<Ctrl-C> Kill inadvertant or hung subprocess

1. Use Sections

If you’re working with blocks of text that are bounded by blank lines, Vi cannot be beat. Navigating through them is just } and {. You can swap two sections in 3 commands:

Command Description
} … Move to the empty line before the 1st section
d } Delete section
} Move past 2nd section
P Paste at current line

Since I use section navigation all the time, I remap my square-brackets to curly-brackets:

map [ {  " All curly-brackets, all the time
map ] }

2. Use Marks

When working with many lines of text, resist the urge to use a count. Instead, use a mark. For example, let’s say you want to cut what appears to be 12 lines of text:

Move to the first line of the block and:

Command Description
m a Mark current position a

Move to the last line of the block and:

Command Description
d ' a Cut to line containing mark a

3. Set Some Options

One of the nice things about Vi is that it’s fairly useable without a lot of tweaking. Nevertheless, here are few settings that can make life better, especially for Vim:

$ vim ~/.vimrc

set nocompatible
set hidden
set title
set ruler
set nohls

4. Map Some Keys

Likewise, the key bindings in Vi are quite good as is. Still, I find adding a few bindings can speed things up nicely. These are my must haves:

$ vim ~/.vimrc

map [ {  " All curly-brackets, all the time
map ] }

nmap <Space> <c-f> " Just like less

nmap <F3>  :w<CR>  " Just like ISPF
nmap <F9>  :q<CR>
nmap <F12> :bd<CR>

5. Find & Replace Like a Boss

Find & Replace is rich and capable, although it works completely differently than what you might expect. In Vi, to change all foo to bar, you do:

Command Description
/ foo <Enter> Find foo
c e bar <Esc> Change to end of word, to the text bar
n Find next
. Repeat last change

The last two commands are repeated as necessary: n . n ..

If you get tired of n . n ., you can Replace All with an Ex command:

Command Description
:%s//bar/g <Enter> Change all occurences of the last search pattern with bar

Vi shines in both the find and replace aspects of this task. Since the find is a regular expression, you can be specific enough to prevent unintended consequences. In the example above, a tighter search pattern would be:

Command Description
/ \<foo\> <Enter> Find the word foo

The \<, \> salad-tongs enforce word boundaries, so food won’t get changed to bard.

Likewise, you can play it safe on the replace side by doing:

Command Description
c 3 l bar <Esc> Change 3 chars forward, to the text bar

although given a tight search string, this is typically not necessary.

Heads Up: One potential annoyance can arise when using the n . n . idiom. When adding text, it pays to search one character before the target and do an append. If you search dead on and do an insert, you’ll need to do n n . n n ., because you haven’t skipped past the original search target.

6. Learn to Move

It’s important to become very good at moving around in a file. Your success with the editor is gated by how quickly and precisely you can move.

Text Movement

Count Cmd Char Mark Description
0 Move to beginning-of-line
$ Move to end-of-line
^ Move to first character of line
<n> w Move forward <n> words
<n> f <c> Move to <n>th occurence of character <c>
<n> t <c> Move just before <n>th occurence of character <c>
; Repeat f, t move to next occurence
, Repeat f, t move to previous occurence
` <m> Move to mark <m>

Line Movement

Count Cmd Char Mark Description
<n> j Move down <n> lines
<n> } Move forward <n> chunks (broken by empty lines)
' <m> Move to line containing mark <m>

Note: The intra-line moves f t ; , provide much of the speed and precision in Vi. It pays to learn and use them.

7. It’s Just Cut, Copy, Paste, & Change

A Vi command can be quite a mouthful, as you can see from the “parts-of-speech” table below.

Name Register Edit Cmd Count Move Cmd Char Mark
Cut " <r> d <n> w t $ ' … <c> <m>
Copy " <r> y <n> w t $ ' … <c> <m>
Paste " <r> p
Change c <n> w t $ ' … <c> <m>

From these elements, we can construct simple commands:

Command Description
y 3 w Copy the next 3 words into the default register

And not-so-simple commands:

Command Description
" a y 2 t ( Copy into register a, the text from the cursor to the 2nd occurence of an open paren

It may seem that Vi has a long list of commands but that’s largely the result of combining the Cut, Copy, Paste, and Change commands with the various Move commands.

For example, guides often list d$ and dw as distinct commands. When in fact, it’s a single delete command combined with two different move commands:

Command Description
d $ Delete to end of line
d w Delete word

You could just as well list these as distinct commands:

Command Description
d t : Delete up to the next colon
d f ; Delete up to and including the next semi-colon

8. Text or Lines: Your Choice

There are text editors (Notepad, Visual Studio, Emacs) and there are line editors (ex, sed, edlin, ISPF Edit). Vi is both.

Vi allows you to choose which form is best for a given action. When working with a stream of words or tokens, you can use text, when working blocks, you can use lines.

For example, let’s say you are in the middle of /etc/fstab and you want to duplicate an existing mount point.

In a garden-variety text editor this would require 6 commands:

Command Description
<Home> Move to column 1
<Shift-End> Select the line
<Ctrl-C> Copy the selected text
<Home> Unselect text
<Down> Move down 1 line
<Ctrl-V> Paste the text

In Vi, you can do it in 2:

Command Description
yy Copy the current line
p Paste the line after the current line

Notice that you can do this from any position in the line. Both the copy and the paste are dealing with lines so the column does not matter. Vi maintains a flag in the copy buffer that tracks whether the buffer contains text or lines. It sets this based on the movement command used to perform the cut or copy. Copy with a text movement and the buffer is marked text. Copy with a line movement and the buffer is marked lines.

Text Commands

Command Description
y $ Copy to end of line
y ^ Copy to first character on line
y ` a Copy up to mark a
y w Copy 1 word
y t ( Copy up to next open paren

Line Commands

Command Description
y j Copy current and next line
y } Copy until next empty line (a section)
y ' a Copy thru line containing mark a
3 yy Copy three lines (not technically a movement)

9. Normal Mode or Cmdline Mode: Your Choice

Even though they seem worlds apart, Normal mode and the Ex Cmdline mode share marks, registers, etc. This lets you perform actions in the environment that is easiest or makes the most sense.

Here’s an example of using marks to bound the start of a sort. I set the mark in Normal mode but then use it in Cmdline mode:

Command Description
m a Mark current position a
} Move to next empty line
:'a,.!sort <Enter> Sort lines from mark a to current line

And here’s an example of collecting text into a register from Cmdline mode and using it in Normal mode:

Command Description
:let @a='' <Enter> Clear register a
:g/set/y A <Enter> For every line that contains set, append to register a
" a P Paste contents of register a

Note: Even though there are 26+ marks a-z, and 26+ registers a-z, I seem to only ever use use mark a and register a. Hmm.

10. Know Your Lists

A good way to explore Vi is by perusing its lists. Note, some of these are only found in Vim.

Name List First Prev Next Last Loading, etc.
Arglist :args :rew :prev :n :last :n *.txt
Buffers :ls :brew :bp :bn :blast :b x.txt, :bd, <Ctrl-^>, :ls!
Marks :marks :m a, :'a
Registers :reg :dis a, :let a=''
Quickfix :cl :crew :cprev :cn :clast :make
Tags :tags :trew :tprev :tn :tlast <Ctrl-]>, <Ctrl-T>, :ta main
Jumps :jumps <Ctrl-O> <Ctrl-I>

There’s a lot more to Vi than I’m covering here, obviously. Hopefully, I’ve provided you with enough tools to get up the learning curve quickly and start reaping its rewards.
