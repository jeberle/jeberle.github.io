DokuWiki

I had the task of choosing a web-based tracking system for what I would call semi-structured data. Generally speaking this was in the realm of a LIMS (Laboratory Information Management System). However, the variation between individual runs was great, and the nature of each run, and the tools around them have a half-life of about 3 months. Therefore, I was loath to invest a lot of time in polishing up a custom-fit solution, only to have it shatter into a million pieces once the next wave of changes blew through.

Given the dynamic nature of the information, I investigated two CMSs (Plone and Drupal) and four Wikis (MoinMoin, TWiki, PmWiki, and DokuWiki). Note, the bar for this task was actually fairly low as the tracking tool this would replace was the mighty Microsoft Excel. Gee, I wonder how many times this scenario has arisen?

Plone seemed a bit too much. I never really bothered trying to get my head around the thing, since again, we’re up against Excel, not, I don’t know, Lotus Notes?!

Drupal was a real gas and I got pretty far with it, after of course, adding the “Date” module, the “CCX” module, the “etc., etc.” module. It’s fun, but ultimately fatiguing. I also worry about a system put together by clicking a great deal. How does one do revision control on the thing?

And so that leaves the Wikis. I’ve done a lot with Python and really like it, so this should have been an easy win for MoinMoin. Something like a prize fighter in a fixed fight. All you have to do is stand there for two rounds. Unfortunately, MoinMoin would have nothing of it. In a fixed fight it finished last. Kinda ugly, OK, but no real categories, namespaces, tags, broken search, bad tables? Man. What is it about web software and Python v. PHP? It’s completely baffling.

Next up, the Perl one. The great one, TWiki. Sub-wikis good, search OK, RCS-based, um, OK. A little hard on the eyes. Some general Perl mojo showing through. Solid B.

Which leads us to the clear winners in the bunch. The ones based on PHP (just like Drupal, WordPress, and MediaWiki). PmWiki is awfully nice. By far the best looking (except tables are little off). Amazing search. The best syntax extensions. One fundamental flaw, however. The pages are stored in text files, that’s good, but all the revisions are stored in the same file. Each revision is a single line with the linefeeds within the bodies escaped as %0A’s. In other words, you’ve got a near binary since you have to filter everything into and out of the storage format! PASS.

Finally, DokuWiki. Really good table handling, good namespaces, nice looking, almost too simple of an architecture, really. It indexes, it renders, it caches, that’s about it. The prize at the bottom of the box? Every page is stored as a text file named pagename + .txt. Want to create 100 pages off-line & feed them to DokuWiki? Just put them in data/pages and run bin/indexer.php, done. Winner: DokuWiki.
