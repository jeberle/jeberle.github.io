Work Rules

  1. Always have something that works
  2. Change one thing at a time
  3. Make sure what you change, changes something
  4. Quick turnaround is essential
  5. If you made 1 mistake, you probably made 2
  6. When in doubt, leave it out
  7. It’s all about finding ways to work faster
  8. Don’t do things the computer can do: math, counting, remembering things, parsing text, etc.
  9. If it takes you 3 hours to incorporate existing code that would have taken you 30 minutes to write from scratch, you’ve got some crappy code on your hands
